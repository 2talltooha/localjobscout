# LocalJobScout

Scrapes 13 Waterloo-region job boards, scores every posting against your resume,
sends desktop notifications for matches, generates tailored cover letters, and
runs an interactive apply pipeline. Runs entirely on your machine.

Built for a first-year pre-medicine student at University of Guelph — but the
config is generic enough to adapt to any applicant profile.

---

## What it does

1. **Scrapes** JobBank, Adzuna, LinkedIn, Indeed, RemoteOK, UGuelph, UWaterloo,
   Laurier, Conestoga, and four hospitals on a schedule (default: every 60 min).
2. **Filters** automatically: strips out-of-province listings, spam companies,
   senior/credentialed roles, and postings requiring more years of experience
   than you have.
3. **Scores** each job against your resume using TF-IDF cosine similarity.
   Jobs above your threshold trigger an OS desktop notification.
4. **Queues** suitable jobs for manual review sorted by match quality.
5. **Generates** tailored cover letters (template or Anthropic API) and validates
   them — fabricated claims like WHMIS, retail experience, or "lab background"
   are flagged before the letter reaches you.
6. **Tracks** applications: mark jobs as interested / applied / interviewed /
   rejected and get reminders on stale follow-ups.
7. **Auto-applies** via email for jobs with a contact address in the description.
   Portal/LinkedIn/Indeed jobs open in your browser for manual submit.

---

## Requirements

- Python 3.11+
- Windows 10 +, macOS 12 +, or Linux with a notification daemon
- ~200 MB disk (mostly the spaCy model)
- Playwright Chromium (~150 MB) for LinkedIn and Indeed scrapers


## Setup

### 1. Put your resume in `data/resume.txt`

Plain text only. If you have a PDF:

```bash
# Use the built-in PDF extractor
# (just point resume_path at the .pdf in config.yaml — pypdf handles it)
```

Or paste from your PDF: open in browser → Select All → paste into a text editor
→ save as `data/resume.txt`.

**Resume quality matters.** Spell out acronyms once ("first aid and CPR") so
the TF-IDF vocabulary captures them.

### 2. Copy and edit `config.yaml`

A minimal working config:

```yaml
location: "waterloo, ON"
match_threshold: 0.22
resume_path: data/resume.txt

prefilter:
  allowed_provinces: ["ON"]
  exclude_min_years_experience: 2

focus:
  keywords: [research, lab, pharmacy, clinical, biology, health, patient, tutor]
```

Full config reference is at the bottom of this file.

### 3. Set API keys (optional but recommended)

Create `.env` in the project root:

```
ANTHROPIC_API_KEY=sk-ant-...
ADZUNA_APP_ID=your_id
ADZUNA_APP_KEY=your_key
```

- **Anthropic key** — enables AI-generated cover letters, interview prep, and
  suitability scoring. Without it the tool still works: cover letters use
  templates, suitability scoring is skipped.
- **Adzuna keys** — free at developer.adzuna.com. Without them the Adzuna
  scraper is silently skipped.

### 4. Check notifications

```bash
python -m localjobscout --check
python -m localjobscout --test-notify
```

A real desktop notification should appear. If `--check` fails, see Troubleshooting.

## Daily workflow

### Step 1 — Scan for new jobs

```bash
python -m localjobscout --once
```

Fetches all sources, deduplicates, scores, and notifies. First run populates
the DB; subsequent runs only process new postings.

To run continuously in the background:

```bash
python -m localjobscout          # runs forever, scans every 60 min
```

### Step 2 — Review the manual queue

```bash
python -m localjobscout --manual-queue
```

Shows the 50–60 suitable jobs sorted by combined match + suitability score.
Each entry shows the title, company, location, scores, URL, and cover letter path.

Open the top 5 in your browser in one go:

```bash
python -m localjobscout --manual-queue --open 5
```

### Step 3 — Generate a cover letter

```bash
python -m localjobscout --cover 9df273eb
```

Use the 8-character ID shown in `--manual-queue`. The letter is saved to
`data/applications/uoguelph-9df273eb.md`. Open it in any editor, review,
and copy-paste into the application form.

If `ANTHROPIC_API_KEY` is set, the letter is generated by Claude and tailored
to the specific job. Otherwise a template is used. Either way, the letter is
validated for fabricated claims before you see it.

### Step 4 — Mark what you applied to

```bash
python -m localjobscout --mark applied 9df273eb
```

Statuses: `seen` · `interested` · `applied` · `interviewed` · `offered` · `rejected` · `hidden`

To remove tracking: `--mark clear <id>`

### Step 5 — Track your pipeline

```bash
python -m localjobscout --applications             # all tracked jobs
python -m localjobscout --applications applied     # just applications sent
python -m localjobscout --follow-up                # applied > 7 days, no update
```

## Auto-apply pipeline

The pipeline does the following for every candidate job:

1. Checks suitability (no spam, no senior roles, no credentials required, Ontario only)
2. Detects application method (email in description → can send; portal/LinkedIn/Indeed → manual)
3. Generates and validates a cover letter
4. Shows you the job + letter for approval (interactive review)
5. For email-method jobs: sends via SMTP with resume attached, marks 'applied'
6. For portal/LinkedIn/Indeed: opens URL in browser, marks 'interested', saves letter
7. Logs everything to `data/auto_apply_log.jsonl`

**Dry run (safe — nothing sent, nothing tracked):**

```bash
python -m localjobscout --auto-apply
python -m localjobscout --auto-apply --auto-apply-score 0.20 --auto-apply-limit 20
```

**Live run with interactive review (recommended for first real use):**

```bash
python -m localjobscout --auto-apply --auto-apply-send
```

For each job you see: title, scores, method, cover letter preview, validation
warnings. Press `s` to send/queue, `k` to skip, `q` to quit.

**Live run without review (careful — only for email-method jobs):**

```bash
python -m localjobscout --auto-apply --auto-apply-send --yes-send-without-review
```

Skips the per-job prompt. Jobs with validation warnings are still auto-skipped.
Everything is logged. Daily limit from config applies.

**To enable email sending**, add to config.yaml:

```yaml
auto_apply:
  enabled: true
  from_email: "you@gmail.com"
  smtp_password: "xxxx xxxx xxxx xxxx"  # Gmail App Password
  daily_limit: 5
```

Note: virtually all Waterloo-area ATS jobs (UGuelph, LinkedIn, Indeed, JobBank
portals) do not have email addresses in the description. Email-send fires mainly
for small employers with direct contact info. The manual-queue and browser-open
path is the real product.

## LLM suitability scoring

If you have an Anthropic API key, score your DB jobs for first-year suitability:

```bash
python -m localjobscout --suitability             # scores 50 jobs
python -m localjobscout --suitability-limit 100   # score more
```

Each job is scored 0–1 with a one-sentence reason ("Entry-level, matches biology
background" vs "Requires MLT credential"). Results are cached in the DB — the API
is never called twice for the same job. Cost: ~$0.001 per job with claude-haiku.

After running `--suitability`, `--manual-queue` will show the suitability score
and reason for each job, and combined scoring (0.6 × match + 0.4 × suitability)
re-ranks the queue.

## Cover letter validation

Every generated letter is checked for claims not supported by the resume:

| Flagged claim | Why |
|---------------|-----|
| WHMIS certification | Not in resume |
| Retail / customer service experience | Not in resume |
| N years of lab/clinical/professional experience | Not in resume |
| Hands-on lab/bench experience | Not in resume |
| Laboratory background / lab experience / bench experience | Not in resume |

In interactive review (`--auto-apply --auto-apply-send`), warnings appear in
red before the send prompt. In unattended mode, jobs with warnings are skipped.

To add your own forbidden phrases without editing code, set in config.yaml:

```yaml
cover_letter:
  forbidden_claims:
    - "prior laboratory work"
    - "strong research background"
```

## Rescore after editing resume

After you update `data/resume.txt`:

```bash
python -m localjobscout --rescore
```

Re-scores every non-excluded job in the DB against the new resume. Also
re-applies prefilter rules, so newly excluded jobs get score = -1.

## Full config reference

```yaml
# ── Core ──────────────────────────────────────────────────────────────────
location: "waterloo, ON"          # passed to JobBank + Adzuna
match_threshold: 0.22             # minimum TF-IDF score for notifications
scan_interval_minutes: 60
use_semantic_matcher: false       # true = sentence-transformers (needs [semantic])
resume_path: data/resume.txt      # .txt or .pdf
db_path: data/jobs.db

# ── Scrapers ──────────────────────────────────────────────────────────────
scrapers:
  jobbank:   { enabled: true,  max_pages: 3, query: "" }
  remoteok:  { enabled: true }
  adzuna:    { enabled: true,  max_pages: 5, query: "" }
  linkedin:  { enabled: true,  max_pages: 3, query: "" }
  indeed:    { enabled: true,  max_pages: 3, query: "" }
  uoguelph:  { enabled: true,  max_pages: 3 }
  uwaterloo: { enabled: true,  max_pages: 3, query: "research assistant" }
  conestoga: { enabled: true,  max_pages: 3 }
  laurier:   { enabled: true,  max_pages: 3 }
  hamiltonhealth: { enabled: false }
  grandriver:     { enabled: false }
  stmarys:        { enabled: false }
  cambridge:      { enabled: false }

# ── Prefilter (applied at scrape time to new jobs) ─────────────────────────
prefilter:
  exclude_phrases: []             # block any job whose title/desc contains these
  exclude_min_years_experience: 2 # 0 = disabled; >N years required → excluded
  allowed_provinces: ["ON"]       # empty = allow all; ["ON"] = Ontario only
  exclude_companies: []           # company name substrings to block
  exclude_title_regex: "^[A-Z]{2,4}\\d{3}"  # blocks Laurier CTF postings

# ── Focus boost (adds to score when keywords found in title/description) ──
focus:
  keywords: [research, lab, laboratory, pharmacy, clinical, biology, health,
             patient, tutor, premed, science, assistant, entry level]
  boost_per_hit: 0.05             # score added per keyword hit
  max_boost: 0.25                 # cap on total boost
  title_boost_multiplier: 2.0     # title hits count 2× vs description hits

# ── Cover letter validation ────────────────────────────────────────────────
cover_letter:
  forbidden_claims: []            # extra phrases always flagged in generated letters

# ── Auto-apply ────────────────────────────────────────────────────────────
auto_apply:
  enabled: false
  from_email: ""
  smtp_host: "smtp.gmail.com"
  smtp_port: 587
  smtp_password: ""               # Gmail App Password — keep in .env
  min_score: 0.22
  daily_limit: 10
  unattended: false               # true = skip per-job review prompt

# ── Email alerts (separate from auto-apply) ───────────────────────────────
alerts:
  enabled: false
  method: email
  email_smtp_host: "smtp.gmail.com"
  email_smtp_port: 587
  email_from: ""
  email_to: []
  email_password: ""
  min_matches_to_alert: 1
```

## Troubleshooting

**No notifications.**
`python -m localjobscout --check` — read the output. On Linux: `pip install dbus-python`.
On Windows: check Focus Assist (Settings → System → Notifications) is off.

**"spaCy model not found."**
`python -m spacy download en_core_web_sm`

**"Playwright not installed" or browser errors.**
`playwright install chromium`

**LinkedIn / Indeed returning 0 jobs.**
These use Playwright with stealth. If they're blocked, LinkedIn returns empty card
pages and Indeed returns a Cloudflare challenge. Both scrapers handle this gracefully
(return `[]`). Try again later — rate limiting is temporary.

**JobBank returning 0 jobs.**
HTML structure occasionally changes. Check the CSS selectors in
`src/localjobscout/scrapers/jobbank.py` against the live page.

**Scores all look low / too many notifications.**
Run `--diagnose` to see your score distribution, then adjust `match_threshold`.
Values 0.20–0.28 are typical. If focus keywords aren't set, add them — they
add up to 0.25 to the base TF-IDF score for relevant jobs.

**Outdated scores after editing resume.**
`python -m localjobscout --rescore`

**Too many unsuitable jobs showing in `--manual-queue`.**
These are pre-existing DB rows from before the province/seniority filters were
added. Mark them `hidden` or delete `data/jobs.db` and re-scan.

**Cover letter has odd keyword phrases** (e.g. "familiarity with pharmacy, assistant").
The focus keyword injection picks literal matches from your config keywords.
Adjust `focus.keywords` to remove short words like "assistant" that match too broadly.
